Pr√°ctico 7: Generadores centrales de patrones (CPGs)#

Un generador central de patrones es un circuito neuronal que puede producir, de forma aut√≥noma, un patr√≥n de actividad, como una oscilaci√≥n, que puede activar los m√∫sculos de manera confiable. Aunque una entrada externa puede encender o apagar el generador central de patrones, o cambiarlo entre modos, y la neuromodulaci√≥n puede ajustar caracter√≠sticas del patr√≥n, el circuito neuronal de un generador central de patrones no depende de ninguna entrada externa para producir el patr√≥n de actividad. Cuando los animales muestran contracciones musculares coordinadas estereotipadas que acompa√±an un comportamiento (como la respiraci√≥n, la locomoci√≥n o la masticaci√≥n), los generadores centrales de patrones suelen estar involucrados.

def I_inj(t, amp):
  """Crea una funci√≥n de corriente inyectada con un pulsos cuadrado.

  Args:
    t (ndarray): vector 1D de tiempos (ms), estrictamente creciente.
    amp (float): amplitud del pulso, en uA/cm^2.

  Returns:
    function: una funci√≥n I(t) que devuelve la corriente inyectada I(t).
  """
  tmax = np.max(t)
  inicios = np.linspace(0, tmax, 8)
  inicio = inicios[1]
  fin = inicios[6]
  
  def I(t):
    return amp * (t > inicio) - amp * (t > fin)

  return I

def I_inj_ruido(t, amp, media=0, desvio=1):
  tmax = np.max(t)
  inicios = np.linspace(0, tmax, 8)
  inicio = inicios[1]
  fin = inicios[6]

  base = amp * (t > inicio) - amp * (t > fin)

  vals = base + np.random.normal(media, desvio, size=t.shape)

  def I(tq):
      return np.interp(tq, t, vals)

  return I

def vdot(v, u, I):
  return (0.04 * v + 5) * v + 140 - u + I

def udot(u, v, a, b):
  return a * (b * v - u)

def step_izhikevich(v, u, I, a, b, c, d, dt):
  spiked = False
  if v < threshold:
    v = v + dt * vdot(v, u, I)
    u = u + dt * udot(u, v, a, b)
  else:
    spiked = True
    v = c
    u = u + d
  return v, u, spiked

Modelo de Izhikevich#

Izhikevich (2003) propuso un modelo simple de neurona que se describe con dos ecuaciones diferenciales acopladas. Una para el potencial de membrana \(v\) y otra para el valor de recobro \(u\):

\[\begin{split}\begin{split} \dfrac{dv(t)}{dt} &= 0.04 v^2(t) + 5 v(t) + 140 - u + I(t) \\ \dfrac{du(t)}{dt} &= a(bv - u) \end{split}\end{split}\]

En conjunto con las condiciones de reestablecimiento:

\[\begin{split}\begin{split} v(v > \theta) &= c \\ u(v > \theta) &= u + d \end{split}\end{split}\]

Este modelo cumple con el requisito de ser computacionalmente eficiente y adem√°s de producir un variado abanico de patrones de disparos observados en neuronas corticales.

Rol de los par√°metros en el modelo de Izhikevich#

El modelo de Izhikevich utiliza cuatro par√°metros principales (a, b, c, d) para modelar una amplia variedad de comportamientos neuronales.

Aqu√≠ se describe el rol de cada par√°metro:

  • a: Controla la velocidad de recuperaci√≥n de la variable de adaptaci√≥n u. Valores bajos de a hacen que la neurona se recupere lentamente despu√©s de un pico, mientras que valores altos producen una recuperaci√≥n r√°pida. Esto influye en la frecuencia de disparo de la neurona.

  • b: Describe la sensibilidad de la variable de adaptaci√≥n u al potencial de membrana v. Valores altos de b hacen que la neurona sea m√°s sensible a los cambios en v, lo que puede producir comportamientos como el rebote post-inhibitorio.

  • c: Representa el potencial de reinicio de la membrana despu√©s de un pico. Este valor determina a qu√© voltaje se restablece la membrana despu√©s de que la neurona ha disparado.

  • d: Controla la recuperaci√≥n de la variable de adaptaci√≥n u despu√©s de un pico. Un valor alto de d provoca un aumento r√°pido en u despu√©s de un pico, lo que puede influir en la duraci√≥n del per√≠odo refractario y la frecuencia de disparo.

En resumen, los par√°metros a, b, c y d permiten ajustar la din√°mica del modelo de Izhikevich para reproducir una amplia gama de comportamientos neuronales, como disparos t√≥nicos, r√°fagas, rebotes post-inhibitorios, y m√°s.

Ajustar estos par√°metros de manera precisa es crucial para simular neuronas espec√≠ficas y para estudiar la din√°mica de las redes neuronales.

Oscilador de medio centro#

El oscilador de medio centro se logra con dos neuronas, cada una de las cuales inhibe a la otra. La actividad de cada neurona oscila en antifase con la otra, de modo que pueden producir contracciones musculares que alternan entre dos m√∫sculos antagonistas. Estas contracciones alternas entre los dos lados del cuerpo de un animal constituyen la unidad b√°sica para la locomoci√≥n mediante deslizamiento o serpenteo, lo cual es com√∫n en animales largos y sin patas (como lampreas, anguilas, serpientes, gusanos, etc.).

Comenzamos definiendo dos constantes:

v0 = -75        # mV
threshold = 30  # mV

Y la funci√≥n de integraci√≥n. Lo importante ac√° es notar como en cada paso, se inyecta corriente en cada neurona m√°s la corriente generada por la otra neurona.

def integrate_izikhevich(t, I1, I2, I_syn_amp, syn_duration, a1=0.02, b1=0.2, c1=-65, d1=2, a2=0.02, b2=0.2, c2=-65, d2=2):
  # Variable de voltaje
  v1 = np.zeros_like(t)
  v2 = np.zeros_like(t)
  v1[0] = v2[0] = v0

  # Variable de recobro
  u1 = np.zeros_like(t)
  u2 = np.zeros_like(t)
  u1[0] = v1[0] * b1
  u2[0] = v2[0] * b2
  
  # Calculamos el valor de salto en el tiempo
  dt = t[1] - t[0]

  I_syn1 = np.zeros_like(t)
  I_syn2 = np.zeros_like(t)

  decay = np.exp(-dt / syn_duration)
  
  # Integramos con el m√©todo de Euler
  for step in range(len(t)-1):
    v1[step+1], u1[step+1], spike1 = step_izhikevich(v1[step], u1[step], I1(t[step]) + I_syn2[step], a1, b1, c1, d1, dt)
    v2[step+1], u2[step+1], spike2 = step_izhikevich(v2[step], u2[step], I2(t[step]) + I_syn1[step], a2, b2, c2, d2, dt)

    if spike1:
      I_syn1[step] += I_syn_amp
    if spike2:
      I_syn2[step] += I_syn_amp

    I_syn1[step + 1] = I_syn1[step] * decay
    I_syn2[step + 1] = I_syn2[step] * decay
      
  return v1, u1, v2, u2, I_syn1, I_syn2

def simulate(amp, syn_duration, I_syn_amp, a1, b1, c1, d1, a2, b2, c2, d2):
    t = np.arange(0, 400, 0.05)
    I1 = I_inj_ruido(t, amp)
    I2 = I_inj_ruido(t, amp)
    V1, u1, V2, u2, I_syn1, I_syn2 = integrate_izikhevich(
        t, I1, I2, I_syn_amp, syn_duration,
        a1, b1, c1, d1,
        a2, b2, c2, d2
    )
    plot_izikhevich(t, I1, I2, V1, u1, V2, u2, I_syn1, I_syn2)

display(widgets.HBox([widgets.VBox([global_controls, neuron1_box, neuron2_box]), widgets.interactive_output(simulate, controls)]))
